2.用户
	2.1 获取短信验证码接口
		POST /api/users/send_auth
		参数
		 mobile 手机

		 返回
		 {
		    error_code
		    error_reason
		    sms_token 验证token
		 }
	2.2 登陆接口
		POST /api/users/login
		参数：
		sms_token 短信验证码 (伴随auth_code提供)
		auth_code 验证码 (与password二选一)
		password 密码（与auth_code二选一）
		mobile 手机号码

		返回
		{
			error_code
			error_reason
			sid 身份信息 必须先更新本地SID
			error_url 跳转地址
		}

	2.4 退出登陆
		请求方式: http-get
		/api/users/logout
		参数

		返回
		{
			error_code  0 成功，非0失败
			error_reason 失败原因
			sid : 设备的SID   123d......
		}

	2.5 用户详细信息
		请求方式: http-get
		/api/users/detail

		{
			error_code  0 成功，非0失败
			error_reason 失败原因
			id 用户id
			sex	性别
			province_id 现居省份
			province_name 省名
			city_id 现居城市
			city_name 城市
			address 常驻地址
			avatar_url 用户头像
			avatar_small_url 用户小头像
			login_name 登录名
			ip_text ip的text
			mobile 手机号
			sid
		}

	2.6 用户信息更新
		/api/users/update

        nickname 昵称
        sex 0:女 1:男
        province_id 省id
        city_id 市id
        address 详细地址
		参数:

		{
			error_code  0 成功，非0失败
			error_reason 失败原因
		}

	2.7 更新push_token

		请求方式：http-post
		/api/users/push_token

		上行参数：
		  push_token
		  push_from 默认为（getui），可以为getui, xinge

		返回结果：
		  {
		    error_code: 0成功，非0失败
		    error_reason: ""
		  }

3.用户收货地址

3.1收货地址
        请求方式: http-get
        /api/user_addresses

        返回
        error_code: 0 成功，非0，失败
        error_reason 失败原因
        user_addresses:[
            {
                id :id
                receiver_name :  收货人
                mobile : 手机号
                province_id ： 省ID
                city_id ： 市ID
                province_name:省名称
                city_name:市名称
                address ： 详细地址
                default_status ： 是否默认地址 1是 0否
            }
        ]

3.2添加/更新收货地址
        请求方式: http-get
        /api/user_addresses/add
        id 地址ID (添加传空 更新必传)
        receiver_name 收货人
        mobile 手机号
        province_id 省ID
        city_id 市ID
        district_id 区ID
        street_address 详细地址
        default_status 是否默认

        返回
        error_code: 0 成功，非0，失败
        error_reason 失败原因

3.3删除收货地址
        请求方式: http-get
        /api/user_addresses/delete
        参数
        id : 地址id

        返回
        error_code: 0 成功，非0，失败
        error_reason 失败原因
